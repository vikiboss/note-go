下面是一份适合高级前端工程师的 12 周 Go 学习计划。它以你已有的 JS/TS/Node.js 经验为起点，重点补齐 Go 的并发、类型系统、工程化与系统编程思维，不局限于 Web。

建议每周投入 8–12 小时：`学 30% + 练 35% + 做 35%`。每周都要有可运行代码、测试和复盘。

## 学习目标

完成后你应能：

- 熟练写出符合 Go 风格的生产代码；
- 理解 goroutine、channel、context、并发控制和竞态；
- 编写 CLI、网络服务、任务系统、数据处理工具；
- 正确使用测试、基准测试、性能分析和调试工具；
- 能阅读常见 Go 开源项目，并独立完成一个中等规模项目。

## 第 0 周：环境与认知迁移

学习：

- 安装 Go、理解 `go mod`、模块版本和工作区。
- 熟悉 `go run`、`go test`、`go vet`、`gofmt`、`go build`。
- 了解 Go 与 Node 的主要差异：
  - 显式错误返回，不依赖异常；
  - 接口隐式实现；
  - goroutine 是轻量并发单元，但不是“自动安全”；
  - 标准库更强，依赖倾向更克制；
  - 组合优于继承。

练习：

- 用 Go 重写三个小 Node 脚本：读取文件、调用 HTTP API、解析 JSON。
- 每个练习至少写一个测试。

产出：

- 一个 `go-learning` 仓库。
- 建议目录：`cmd/` 放入口、`internal/` 放内部逻辑、`pkg/` 只放确实可复用的公共库。

## 第 1–2 周：Go 基础与惯用写法

学习：

- 基本类型、零值、数组、slice、map、字符串与 rune。
- 函数、多返回值、可变参数、defer。
- struct、方法、指针接收者和值接收者。
- interface、组合、嵌入。
- error 的创建、包装、判断与处理。
- 包、可见性、模块管理。

重点转换：

- TS 的 interface 是“先声明契约”；Go 的 interface 通常应由使用方定义，保持小而专。
- 少写“万能工具类”；多写小函数、小接口和组合结构。
- 不要急于泛型化；先写清晰的具体实现。

练习：

- 实现一个内存版 Todo/书籍/订单管理库：
  - CRUD；
  - 查询、排序、分页；
  - 自定义错误；
  - JSON 导入导出。
- 每个核心函数写表驱动测试（table-driven tests）。

验收：

- `go test ./...`
- `go vet ./...`
- `gofmt -w .`

## 第 3 周：测试、调试与代码质量

学习：

- `testing` 标准库、子测试、表驱动测试、测试辅助函数。
- mock 的适用边界：优先通过小接口和依赖注入测试，少用复杂 mock。
- benchmark、race detector、覆盖率。
- `pprof` 基础。
- lint、错误处理规范、日志规范。

练习：

- 为前两周项目补齐：
  - 边界测试；
  - 错误路径测试；
  - benchmark；
  - 一个故意制造的数据竞争，再用 `go test -race` 定位。

验收命令：

```bash
go test ./...
go test -race ./...
go test -cover ./...
go test -bench=. ./...
```

## 第 4–5 周：并发编程

学习：

- goroutine、channel、select。
- `sync.Mutex`、`RWMutex`、`WaitGroup`、`Once`、`Pool`、`atomic`。
- context 的取消、超时、传播。
- worker pool、fan-out/fan-in、限流、背压。
- 常见问题：goroutine 泄漏、死锁、数据竞争、channel 滥用。

核心原则：

- “通过通信共享内存”是建议，不是教条；共享状态用 mutex 往往更直白。
- channel 更适合任务流、事件流、所有权转移；mutex 更适合保护状态。
- 所有可能长时间运行的任务都应考虑 `context.Context`。

练习：

- 实现并发文件处理器：
  - 遍历目录；
  - 限制并发数；
  - 支持取消；
  - 汇总成功、失败与耗时；
  - 检测重复文件内容（可使用哈希）。
- 再实现一个 URL 抓取器，但重点放在并发、重试、超时、限流，而不是 Web 框架。

验收：

- `go test -race ./...` 无竞态。
- 主动取消时能快速退出，无明显 goroutine 泄漏。

## 第 6 周：CLI 与系统工具

学习：

- `flag` 或 Cobra 的 CLI 设计。
- 标准输入输出、退出码、环境变量、配置文件。
- 文件系统、路径、进程执行、信号处理。
- `io.Reader` / `io.Writer`：这是 Go 非常重要的抽象。

练习：

- 做一个真正可用的 CLI，选一个：
  - 日志分析器；
  - 批量图片/文件整理工具；
  - Git 提交信息统计工具；
  - JSON/CSV 转换与校验工具；
  - 本地目录增量备份工具。

要求：

- 支持 `--help`；
- 支持 stdin/stdout；
- 有明确退出码；
- 有单元测试；
- 发布一个跨平台二进制文件。

## 第 7 周：网络与协议，不只做 Web

学习：

- `net/http` 的客户端与服务端。
- HTTP 中间件、超时、连接复用。
- TCP/UDP 基础、`net` 包。
- JSON、CSV、二进制编码的取舍。
- RPC、消息队列、WebSocket 的概念和适用场景。

练习：

- 实现一个 TCP 多人聊天室，或一个简单的键值服务：
  - 自定义简单协议；
  - 多客户端并发；
  - 心跳/超时；
  - 优雅关闭；
  - 基础压测。

这周的目标是理解网络 I/O 和协议边界，而不只是“起一个 API”。

## 第 8 周：数据、存储与后台任务

学习：

- `database/sql` 的连接池和上下文取消。
- SQL 基础、事务、索引、分页、迁移。
- 缓存、队列、幂等、重试、死信的工程概念。
- 文件存储与流式处理。

练习：

- 将前面的一个项目接入 SQLite 或 PostgreSQL。
- 增加：
  - 数据迁移；
  - 事务；
  - 任务队列或定时任务；
  - 幂等处理；
  - 失败重试。

## 第 9 周：性能与可观测性

学习：

- CPU/内存/阻塞 profile。
- benchmark 的正确使用。
- GC、逃逸分析、分配优化的基本判断。
- 结构化日志、指标、trace 的基本理念。
- 不要过早优化：先测量，再判断，再修改，再验证。

练习：

- 为已有项目构造一个性能瓶颈。
- 用 benchmark 或 pprof 定位。
- 优化前后记录数据，写一份简短性能报告。

常用工具：

```bash
go test -bench=. -benchmem ./...
go test -cpuprofile cpu.out ./...
go tool pprof cpu.out
go build -gcflags="-m" ./...
```

## 第 10–12 周：综合项目

选一个偏非 Web 的主项目，做完整工程闭环。

推荐方向：

| 项目 | 能练到的能力 |
|---|---|
| 本地优先的文件同步工具 | 文件系统、哈希、并发、断点续传、CLI |
| 轻量任务调度器 | 并发、队列、重试、持久化、可观测性 |
| 日志采集与分析管道 | 流式 I/O、解析、批处理、并发、性能 |
| P2P 局域网文件传输工具 | TCP/UDP、协议设计、并发、校验 |
| 简易消息队列 | 网络协议、持久化、consumer group、背压 |
| 代码仓库分析工具 | Git、CLI、AST、缓存、并发任务 |

建议选择“本地优先的文件同步工具”，因为它覆盖 Go 最有价值的部分，又不会被 Web 框架带偏。

最终项目最低要求：

- 有清晰 README 与架构说明；
- 有 CLI 或稳定 API；
- 关键逻辑单元测试；
- `go test -race ./...` 通过；
- 支持 context 取消和优雅退出；
- 有结构化日志；
- 至少做过一次基准测试或 pprof 分析；
- 使用 GitHub Actions 或类似 CI 自动跑格式化、测试、静态检查。

## 每周固定节奏

- 周一至周二：学习语言与标准库，做小实验。
- 周三至周四：完成 3–5 个针对性练习。
- 周五至周日：推进项目、补测试、重构、写复盘。

每周复盘四件事：

1. 本周最容易按 JS 思路写错的地方是什么？
2. 哪个标准库包值得深入？
3. 哪段代码在并发、错误处理或接口设计上不够 Go？
4. 下周项目要交付什么可运行结果？

## 学习资料优先级

按这个顺序即可：

1. [A Tour of Go](https://go.dev/tour/)
2. [Effective Go](https://go.dev/doc/effective_go)
3. [Go by Example](https://gobyexample.com/)
4. [Go Blog](https://go.dev/blog/)
5. 《The Go Programming Language》
6. 《Concurrency in Go》
7. 阅读标准库源码：`context`、`io`、`net/http`、`sync`、`encoding/json`。

最后一个建议：前端背景的人很容易快速掌握 Go 语法，但真正拉开差距的是并发边界、I/O 抽象、错误处理和可观测性。把至少一半时间投入到这些地方，比连续刷算法题更有效。
