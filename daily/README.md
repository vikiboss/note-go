# Go 13 周学习实验室

这里是 [总学习计划](../plan.md) 的可执行部分。每个日期目录代表从该周一开始的一周，是独立 Go module；进入目录后即可运行，不需要把所有周次一次性学完。书籍版本、阅读程度和使用方法见 [阅读指南](reading-guide.md)。

## 如何使用

1. 先读本周 README，回答“开始前诊断”，不要先看测试实现。
2. 运行现有程序与测试，记录你对输出和失败路径的预测。
3. 按“基础 → 标准 → 挑战”推进；时间不足时至少完成基础档。
4. 每次修改都补一个能证明行为的测试。
5. 周末执行验收命令，把命令、结果和复盘写进自己的学习日志。

示例不是“最终最佳实现”，而是这一周适合研究的最小基线。遇到 TODO 或加餐时，应先写验收测试，再改实现。

## 统一质量门槛

每周至少执行：

```bash
gofmt -w .
go test ./...
go vet ./...
```

并发周额外执行 `go test -race ./...`；性能周执行 `go test -bench=. -benchmem ./...`。如果受限环境不能写默认构建缓存，可临时设置 `GOCACHE=/tmp/go-learning-cache`。

## 学习记录建议

每周留下四类证据：一段可运行代码、一个你新增的失败/边界测试、一条工具输出（test/race/benchmark/profile）、一段 150–300 字复盘。不要只记录“学了什么”，要记录“原先判断、实验结果、修正后的模型”。
